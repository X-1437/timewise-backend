# 数据结构设计 - 初稿

---

## 📖 完整项目例子：个人笔记应用

我们用一个**最简单的笔记应用**来完整演示数据结构设计！

### 功能
- 创建笔记
- 查看笔记列表
- 查看单条笔记
- 删除笔记

---

## 第一步：确定有哪些功能

```
1. 创建笔记（输入标题和内容，保存到数据库）
2. 查看笔记列表（把所有笔记展示出来）
3. 查看单条笔记（点进去看详细内容）
4. 删除笔记（把笔记删掉）
```

---

## 第二步：确定有哪些数据

根据功能，我们需要存：

```
笔记（Note）
- 标题
- 内容
- 创建时间
```

就一个数据！太简单了！

---

## 第三步：设计数据结构（Schema）

```javascript
// 笔记表（MongoDB）
{
  "_id": ObjectId("xxx"),    // 笔记ID，主键
  "title": "我的第一篇笔记",  // 标题
  "content": "今天天气真好",  // 内容
  "createdAt": "2026-06-02T10:00:00Z",  // 创建时间
  "updatedAt": "2026-06-02T10:00:00Z"   // 更新时间
}
```

---

## 第四步：设计索引（让查询更快）

```javascript
// 按创建时间倒序索引（查看列表时按最新排序）
db.notes.createIndex({ "createdAt": -1 })
```

---

## 第五步：总结设计要点

1. **一个实体（Note）**：因为功能简单
2. **基本字段**：id, title, content, createdAt, updatedAt
3. **索引优化**：按时间排序

## 回到我们的项目：时间序列分析助手

用同样的方法！

## 1. 功能梳理

```
1. 会话管理
   - 创建会话
   - 查看会话列表
   - 删除会话

2. 消息管理
   - 发送消息
   - 查看消息历史

3. 文件管理
   - 上传CSV文件
   - 解析CSV并存到数据库

4. 聊天功能
   - 用户发消息
   - 大模型回复
   - 调用MCP工具
```

---

## 2. 实体梳理

根据功能，我们有4个实体：

```
1. Session（会话）
2. Message（消息）
3. File（文件）
4. TimeSeriesData（时间序列数据）
```

---

## 3. 实体关系

```
Session ←1----n→ Message
Session ←1----n→ File
File ←1----n→ TimeSeriesData
```

---

## 4. 详细数据结构

### 4.1 Session（会话）

```javascript
{
  "_id": ObjectId("xxx"),
  "title": "2026-06-02 销售分析",
  "createdAt": "2026-06-02T10:00:00Z",
  "updatedAt": "2026-06-02T10:30:00Z"
}
```

**索引**：
```javascript
db.sessions.createIndex({ "updatedAt": -1 })
```

---

### 4.2 Message（消息）

```javascript
{
  "_id": ObjectId("xxx"),
  "sessionId": ObjectId("会话ID"),
  "role": "user",
  "content": "分析一下这份数据",
  "timestamp": "2026-06-02T10:00:00Z",
  "fileId": ObjectId("文件ID"),  // 可选
  "toolCalls": [  // 可选
    {
      "toolName": "eda_analysis",
      "toolArgs": {},
      "toolResult": "分析结果...",
      "timestamp": "2026-06-02T10:01:00Z"
    }
  ]
}
```

**索引**：
```javascript
db.messages.createIndex({ "sessionId": 1, "timestamp": 1 })
```

---

### 4.3 File（文件信息）

```javascript
{
  "_id": ObjectId("xxx"),
  "filename": "sales.csv",
  "size": 102400,
  "rowCount": 1000,
  "columnNames": ["date", "sales", "product"],
  "uploadedAt": "2026-06-02T10:00:00Z",
  "sessionId": ObjectId("会话ID")
}
```

**索引**：
```javascript
db.files.createIndex({ "sessionId": 1 })
```

---

### 4.4 TimeSeriesData（时间序列数据）

**创建时间序列集合**：
```javascript
db.createCollection("time_series_data", {
  timeseries: {
    timeField: "timestamp",
    metaField: "metadata",
    granularity: "hours"
  }
})
```

**数据结构**：
```javascript
// 假设CSV内容：
// date, sales, product
// 2026-05-01, 1000, A

{
  "timestamp": "2026-05-01T00:00:00Z",
  "metadata": {
    "fileId": ObjectId("文件ID"),
    "product": "A"
  },
  "sales": 1000
}
```

**索引**：
```javascript
db.time_series_data.createIndex({ "metadata.fileId": 1 })
```
