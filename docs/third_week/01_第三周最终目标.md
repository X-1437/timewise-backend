## 四、验收标准

### 4.1 功能验收
- [ ] MongoDB服务正常运行，后端能成功连接
- [ ] 会话数据持久化到MongoDB，刷新页面后历史会话不会丢失
- [ ] 消息数据持久化到MongoDB，刷新页面后历史消息不会丢失
- [ ] 文件元数据持久化到MongoDB
- [ ] 支持新建会话、获取会话列表、切换会话、删除会话
- [ ] 配置OpenAI API Key后，能调用真实大模型
- [ ] 真实大模型能识别用户意图并调用MCP工具（eda_analysis/preprocessing/feature_analysis/forecast）
- [ ] MCP工具能正常返回分析结果
- [ ] 完整流程可跑通：上传CSV → 创建会话 → 发送消息 → 大模型调用MCP工具 → 查看结果

### 4.2 技术验收
- [ ] 使用PyMongo作为MongoDB驱动
- [ ] 数据模型设计合理，包含必要的索引
- [ ] API接口符合RESTful规范
- [ ] 有基本的错误处理和日志记录
- [ ] 保留规则匹配作为降级方案（无API Key时使用）
- [ ] 代码结构清晰，易于后续扩展
