# 大模型与意图识别配置说明（第四周）

本说明用于解释第四周“意图识别 -> 工具调用”的技术口径，以及本地环境如何配置 DeepSeek（OpenAI SDK 兼容）。

## 1. 第四周的意图识别职责边界

- 用户输入：任意自然语言（不依赖固定句式）
- 大模型职责：识别意图并选择正确的工具（tool choice）
- 系统职责：补齐 `session_id/file_id` 等上下文参数，并执行 MCP 工具

对应代码入口：

- 意图识别：[LLMClient](file:///d:/trae_workspace/data/file/fourth_week/code/llm/client.py)
- 参数补齐与 MCP 调用：[ChatService](file:///d:/trae_workspace/data/file/fourth_week/code/services/chat_service.py#L42-L71)

## 2. 使用 DeepSeek 的方式

第四周使用 OpenAI Python SDK（兼容模式）对接 DeepSeek：

- `OPENAI_BASE_URL` 指向 DeepSeek 的兼容地址
- `OPENAI_MODEL` 指向 DeepSeek 模型名
- `OPENAI_API_KEY` 为你的 key

配置读取位置：

- `fourth_week/code/config.py` 会优先读取同目录下 `.env` / `.env.example`（见 [config.py](file:///d:/trae_workspace/data/file/fourth_week/code/config.py)）

## 3. 本地配置注意事项

1. 为避免 key 泄露，本仓库文档中不记录真实 key
2. `.env` 只用于本机运行，不应进入任何对外提交动作
3. 若大模型不可用，系统会自动降级为规则兜底（见 [LLMClient.analyze_intent](file:///d:/trae_workspace/data/file/fourth_week/code/llm/client.py#L16-L18)）

## 4. “工具提示未配置”的说明

在 IDE 的集成层面，可能会提示 openai provider 未配置；这属于“IDE 集成配置”层面的提示，不影响后端代码从 `.env` 读取并直接调用模型。

如果你希望 IDE 工具链也能识别配置，需要在 IDE 的集成设置里补齐对应 provider 的配置项（不在代码层面处理）。
