## 3. 开发与验证的“统一输出格式”（严格执行）

所有“分析类结果”统一输出为：

- 输出：图片（PNG） + 打印信息（代码块） + 必要的表格（Markdown table）
- 结论：只写事实结论与可验证结论

## 3.1 图片产出与命名规范（方案 A，已确认）

图片生成方式
- 后端生成 PNG（方案 1）：由后端在执行 MCP 工具时生成图表 PNG，落盘到后端 `evidence/` 目录，并在 MongoDB `artifacts` 集合记录元数据
- Markdown 引用方式：步骤输出的 Markdown 中以 `![](/api/v1/...)` 引用图片下载链接；右侧预览与导出报告复用同一份 Markdown，保证一致性

命名与可追溯策略
- 对用户可见（Markdown 标题/说明）：使用中文语义名，例如“EDA-按月可视化”“预处理-噪音识别”
- 对系统落盘（PNG 文件名）：使用安全 ID（避免中文进入 URL 造成编码与兼容问题）
- 对 MongoDB 记录：除 `type/path/sessionId/fileId/createdAt` 外，新增记录中文语义名字段（例如 `displayName`）

### 3.2 出图清单（验收门槛）

按 `D:\trae_workspace\data\file\fifth_week\ex\` 示例冻结为最低出图标准（缺图即不通过 UAT）：

- EDA：可视化至少 1 张
- 预处理：噪音识别至少 1 张；异常值处理至少 1 张
- 特征提取：
  - 趋势至少 1 张
  - 时域特征：周期性/相关性/季节性至少各 1 张
  - 频域特征：周期性/傅里叶变换/小波变换至少各 1 张
- 朴素预测：根据用户选择的方法（相邻两行/相同时间点/按天累加）至少输出 1 张对应图

#### 4.1.5 边界与兜底策略落地（必须可测）

兜底场景与输出模板（以服务端为最终兜底）
- 统一原则：只要输入与数据分析无关，统一返回同一类引导性回复
- 统一回复结构：
  - 身份说明：当前是时间序列数据分析助手
  - 能力边界：当前支持的数据分析范围
  - 下一步引导：提示用户上传数据或明确分析目标
- 特殊说明（可选附加句）：
  - 若输入涉及实时外部信息，可补一句“当前无法直接获取该类实时数据”
  - 若输入涉及未知算法，可补一句“当前没有对应算法工具”
- 无论是否附加说明，都不得改变“统一引导回复”这一主结构

执行要求
- 当模型未返回 tool_calls 或返回未知工具名时，服务端必须走固定兜底模板，避免输出不确定/编造结果
- 分析工具链路必须改为“白名单准入”：只有明确命中 EDA / 预处理 / 特征 / 预测 / 导出意图时才允许调用工具
- 默认回落策略必须从“EDA”改为“统一引导回复”，避免把无关输入误解释为分析请求
- 将“兜底输出”纳入 UAT 用例（见 5.1）

实施计划（本轮仅先写入方案，待确认后再继续改代码）
1) 梳理现有输入分流规则，识别“默认回落到 EDA”的具体入口
2) 将输入判断改为“分析意图白名单”而不是“若干非分析关键词黑名单”
3) 定义统一引导回复模板，不再为每类非分析输入设计独立主路径
4) 在服务端增加最终兜底：
   - 若不属于明确分析意图，则返回统一引导回复
   - 禁止默认进入 EDA
5) 增补 UAT 用例池，验证“任意非分析输入”都不会误触发分析工具

#### 4.1.6 标准主流程一键运行（本轮先写入方案，待确认后再改代码）

目标
- 用户输入“帮我运行全流程”，系统自动执行：数据上传确认 → EDA → 预处理 → 特征提取 → 朴素预测（三种方法） → 导出报告
- 该能力属于分析意图白名单，与 EDA/预处理/特征/预测/导出同级

MVP 固定参数（与 SRS 保持一致）
- EDA：原始数据
- 预处理：异常值处理（outlier_remove）
- 特征/预测：预处理后数据
- 朴素预测顺序：相邻两行 → 相同时间点 → 按天累加
- 预测步数：7

实现要求（研发落地口径）
- 服务端编排执行：按固定顺序串行调用既有工具，复用单步工具的输出（保证一致性）
- 过程可见性：每一步开始/完成/失败都要可感知（对话输出或预览区状态均可）
- 失败处理：任一步失败则停止后续步骤，并返回“失败步骤 + 原因摘要 + 下一步建议（重试/先补前置步骤/换参数）”
- 导出交付：全流程成功后自动触发导出；导出默认离线包（ZIP），解压后本地打开 Markdown 可直接看到图片
- 可扩展性：后续可增加按钮入口与可选参数，但不影响 MVP 的固定流程

#### 4.2.1 全链路算法校验（清单化执行）

校验原则

误差指标口径（默认建议，可在校验记录中固定）
