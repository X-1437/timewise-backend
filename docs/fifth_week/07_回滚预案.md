## 回滚触发条件

以下任一条件触发立即回滚：

| 编号 | 触发条件 | 阈值 |
|---|---|---|
| R-01 | 核心功能不可用 | EDA / 预处理 / 特征 / 预测 / 导出任一步骤无法正常完成 |
| R-02 | 错误率异常 | 5 分钟内错误率 > 20% |
| R-03 | 性能严重下降 | 端到端全流程 > 60s（阈值的 4 倍） |
| R-04 | 图片/导出链路中断 | 连续 3 次导出 ZIP 内无图片或损坏 |
| R-05 | 数据库连接中断 | MongoDB 连续 60s 不可用 |

---

## 回滚步骤

1. **停止当前后端进程**
   - `taskkill /F /PID <当前后端PID>`
2. **切回第四周稳定版代码**
   - 当前运行代码位于 `mcp/fourth_week/code/`
   - 回滚时使用 Git 上一个稳定 tag（如 `v0.4.0`）
3. **清理第五周新增 artifacts（可选）**
   - 删除 `evidence/` 下回滚窗口期内生成的新图片
   - 或保留（不影响旧功能）
4. **重启后端**
   - `python backend.py`
5. **验证健康检查**
   - `curl http://127.0.0.1:8000/api/v1/sessions` 返回正常
6. **通知相关方**
   - 说明回滚原因和时间窗口
