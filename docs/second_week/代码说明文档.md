## 六、核心文件详细说明

### 6.1 App.tsx（前端核心）

**作用**：主应用组件，管理整个应用的状态

**核心功能**：
- `messages` 状态：存储所有聊天消息
- `uploadedFileId` 状态：记录当前上传的文件ID
- `handleSendMessage()`：处理发送消息
- `handleFileUpload()`：处理文件上传

### 6.2 api.ts（API封装）

**作用**：封装与后端的HTTP请求

**核心接口**：
- `api.uploadFile(file)`：上传CSV文件
- `api.sendMessage(content, fileId)`：发送消息

### 6.3 backend.py（后端核心）

**作用**：FastAPI应用，提供REST API

**核心接口**：
- `POST /api/v1/files/upload`：文件上传
- `POST /api/v1/chat/messages`：消息发送

---

**💡 什么是REST API？**

REST API是大模型对话产品的**主流方式**！几乎所有大模型产品都是这样的架构：

```
┌─────────────────┐
│   前端界面      │ ← React/Vue/原生JS（你看到的页面）
└────────┬────────┘
         │ HTTP请求
         ↓
┌─────────────────┐
│   后端API       │ ← FastAPI/Flask/Django（REST API）
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│   大模型API     │ ← OpenAI/通义千问
└─────────────────┘
```

常见的大模型产品都是这样做的：
- ✅ ChatGPT网页版
- ✅ 文心一言
- ✅ 通义千问
- ✅ 你之前的Web版也是这样的架构

### 6.4 rule_matcher.py（规则匹配）

**作用**：模拟大模型回复，不用花钱买token

**支持的问题**：
1. "这个CSV里有什么数据"
2. "帮我看看数据概览"
3. "上传成功了吗"
4. "你好"

## 六、前后端交互流程

```
1. 用户点击"选择CSV文件"
   ↓
2. FileUpload组件触发上传
   ↓
3. 前端调用 api.uploadFile()
   ↓
4. 后端接收文件，保存到 temp_uploads/
   ↓
5. 后端返回文件ID
   ↓
6. 用户输入问题并发送
   ↓
7. 前端调用 api.sendMessage()
   ↓
8. 后端rule_matcher匹配规则
   ↓
9. 后端返回回复
   ↓
10. 前端展示消息
```

**💡 什么是ASGI服务器Uvicorn？**

简单理解：**Uvicorn是后端的Web服务器**

| 类比 | 说明 |
|------|------|
| **前端开发服务器** | Vite（让你能访问 http://localhost:5173） |
| **后端Web服务器** | Uvicorn（让你能访问 http://localhost:8000） |

更详细的解释：

| 名词 | 作用 |
|------|------|
| **FastAPI** | Web框架，帮你写API逻辑（比如 `/api/v1/files/upload`） |
| **Uvicorn** | ASGI服务器，把FastAPI跑起来，让浏览器能访问 |

**类比**：
- FastAPI = 餐厅的菜单和厨师（决定做什么菜）
- Uvicorn = 餐厅的服务员和店面（让客人能进来点菜）
