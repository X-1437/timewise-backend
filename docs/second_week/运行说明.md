# 第二周项目运行说明

## 一、项目结构

```
second_week/
├── frontend/          # 前端代码（React+TypeScript）
├── code/              # 后端代码（Python+FastAPI）
├── test_data/         # 测试数据
└── *.md               # 文档
```

## 二、运行步骤

### 步骤1：启动后端服务

```bash
# 1. 进入后端目录
cd D:\trae_workspace\data\file\second_week\code

# 2. 激活conda环境
conda activate hongsu-mcp

# 3. 安装依赖（如果还没安装）
pip install -r requirements.txt

# 4. 启动后端服务
python backend.py
```

后端服务会在 `http://localhost:8000` 启动。

### 步骤2：启动前端服务

打开新的终端窗口：

```bash
# 1. 进入前端目录
cd D:\trae_workspace\data\file\second_week\frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

前端服务会在 `http://localhost:5173` 启动。

### 步骤3：在浏览器中打开

访问 `http://localhost:5173` 即可看到应用。

## 三、测试方法

### 测试1：文件上传功能
1. 点击"📁 选择CSV文件"按钮
2. 选择 `test_data/test_sales.csv`
3. 应该看到"文件上传成功"提示

### 测试2：左侧侧边栏功能
1. 点击顶部Header左侧的"◀"或"▶"按钮 → 侧边栏应该收起/展开
2. 点击"新建会话"按钮 → 应该创建新会话并自动切换
3. 点击侧边栏上的会话 → 应该切换到对应的会话

### 测试3：规则匹配对话
上传文件后，可以测试以下问题：
1. "这个CSV里有什么数据"
2. "帮我看看数据概览"
3. "上传成功了吗"
4. "你好"

## 四、API接口说明

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/files/upload` | POST | 上传CSV文件 |
| `/api/v1/chat/messages` | POST | 发送消息，获取回复 |

## 五、注意事项

1. 确保后端和前端都在运行
2. 后端端口是8000，前端端口是5173
3. 上传的文件会保存到 `code/temp_uploads/` 目录
4. 目前使用规则匹配，不需要买token
