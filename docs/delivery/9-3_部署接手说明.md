## 6. 启动方式

### 6.1 后端启动

进入目录：

```powershell
cd d:\trae_workspace\data\github\timewise-backend\mcp\src
```

安装依赖：

```powershell
pip install -r requirements.txt
```

启动服务：

```powershell
python backend.py
```

或使用：

```powershell
uvicorn backend:app --host 0.0.0.0 --port 8000
```

启动成功后，默认访问地址为：
- `http://localhost:8000`

接口文档可访问：
- `http://localhost:8000/docs`

### 6.2 前端启动

进入目录：

```powershell
cd d:\trae_workspace\data\github\timeswise
```

安装依赖：

```powershell
npm install
```

启动开发服务：

```powershell
npm run dev
```

如只做构建验证，可执行：

```powershell
npm run build
```

### 6.3 启动顺序建议

建议按以下顺序启动：

第一步：
- 先确认 MongoDB 服务可用

第二步：
- 启动后端 `mcp/src`

第三步：
- 打开后端文档页确认服务正常

第四步：
- 启动前端 `timeswise`

第五步：
- 在浏览器中完成最小验收

## 9. 接手时常见误区

接手过程中最容易出现以下误区：

误区一：
- 把 `app/` 当成当前正式后端目录

正确做法：
- 当前正式后端目录是 `mcp/src`

误区二：
- 认为只有代码需要交付，第一周到第五周文档不重要

正确做法：
- 五周 `md` 文档属于正式交付物，是项目过程和闭环的重要依据

误区三：
- 把 `mcp/` 下各周目录直接当成当前正式部署目录

正确做法：
- 各周目录是阶段性记录
- 正式部署目录以 `mcp/src` 为准
